# 高级 A+ 自动化 PoC — 技术方案

> 状态:方案设计稿(未写代码、未装依赖)
> 适用环境:本机 Mac(darwin),PoC 运行机器
> 最后更新:2026-06-26

---

## 1. 目标与范围

在亚马逊**多站点**场景下,通过**紫鸟浏览器 WebDriver** 驱动卖家后台,实现**高级 A+(Premium A+)** 内容的:

1. **自动判断**(四项):资格/现状、模板/模块选择、素材齐全性、内容查重/合规
2. **自动制作与上传**:在后台 A+ Content Manager 里创建/填充高级 A+ 并提交

本文件先定方案;交付节奏:**先做 PoC**(最小可跑通,验证可行后再扩)。

---

## 2. 关键技术结论(为什么走 UI 自动化)

- **SP-API 不支持创建高级 A+。** A+ Content API 仅覆盖**标准 A+**(contentDocument 增删改),高级/Premium A+ 受品牌资格门控、模块未通过 API 开放,只能在卖家后台 UI 完成。
  - ⚠️ 待复核:以亚马逊官方最新文档为准(本结论需在 PoC 启动前再核一次,避免 API 已扩能)。
- 因此**浏览器 UI 自动化是唯一现实路径**。多账号/多站点 + 风控考虑下,**紫鸟浏览器(防关联环境)+ WebDriver 接管**优于普通 Selenium 直连。

## 4. 紫鸟 WebDriver 接管机制

来源:官方 `ziniao_webdriver_demo`(纯 Python)。

**已知:**
- 通过 `config.yaml` 配置:`browser.client_path`(紫鸟客户端路径)、`browser.webdriver_path`(ChromeDriver 路径)。
- 任务函数签名:`def task(driver, store_name: str, download_path: str)`,`driver` 即接管后的 Selenium WebDriver。
- 框架用 `func_dict` 映射任务名→函数,`use_one_browser_run_task` 管理"开窗口→跑任务→收尾"流程。
- 客户端路径按平台不同:Mac 为 `ziniao`。

**待确认(需读 demo 源码 / requirements.txt,PoC 第一阶段产出):**

- [ ] Selenium 接管方式(`debuggerAddress` 还是直接由 demo 框架返回 driver)
- [ ] ChromeDriver 版本如何与紫鸟内核版本对齐
- [ ] 仓库 License(GitHub 页未显式标注)

## 5. 自动判断引擎(四项)

| 判断项 | 判断内容 | 数据来源 | PoC 可达性 |
|---|---|---|---|
| **资格/现状** | 该 ASIN/品牌是否有高级 A+ 资格;是否已有 A+;是否需升级(标准→高级) | A+ Content Manager 列表页 + 品牌资格状态 | ✅ 优先实现 |
| **模板/模块选择** | 该用哪套高级 A+ 模板/模块组合 | 规则表(品类→模板映射,需你提供业务规则) | ⚠️ 依赖业务规则 |
| **素材齐全性** | 图片/文案/视频是否齐备、尺寸是否合规(高级 A+ 对模块素材有硬性尺寸要求) | 素材库(来源待定)+ 高级 A+ 模块规格 | ⚠️ 依赖素材来源 |
| **内容查重/合规** | 文案是否违反亚马逊规则、是否与现有内容重复 | 规则/关键词清单 + 现有内容比对 | ⚠️ 规则需明确 |

**未决输入(需你给):** 模板映射规则、素材来源/命名约定、合规关键词清单。

## 6. 高级 A+ 制作 & 上传流程(后台 UI)

后台路径:Seller Central → 广告/品牌 → **A+ Content Manager** → 创建 → 选**高级 A+ 模块** → 填素材/文案 → 选 ASIN → 提交审核。

**多站点考量:**
- 每个站点是独立的紫鸟店铺窗口 + 独立后台 + 独立语言/文案。
- 同一品牌跨站点可"克隆"内容,但语言、合规、ASIN 各站点不同 → 需站点维度的素材/文案矩阵。

> ⚠️ 后台具体 DOM 选择器/步骤**不预先臆造**,在 PoC 搭建时对真实后台逐步抓取并固化(选择器脆弱,需做容错与显式等待)。

## 9. 风险与未决项

- **合规/ToS**:自动化操作的是你自己店铺的自有 listing 内容,属正常卖家运营;但需控制频率、做人审兜底,避免触发风控。仅自动到"草稿"、提交保留人工确认更稳。
- **选择器脆弱**:后台 UI 改版会断;需显式等待 + 容错 + 失败截图。
- **紫鸟接管机制未确认**(§4 待确认项)——M0 落实前不写死。
- **素材/模板/合规规则未提供**——判断项②③④阻塞在此。
- **高级 A+ 资格**:无品牌资格则后台根本没有高级 A+ 入口,自动化无从谈起——需先确认资格。
